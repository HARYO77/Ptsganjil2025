(1).CRUD adalah serangkaian fungsionalitas mendasar yang biasanya digunakan dalam manajemen database. CRUD mencakup fungsi Create, Read, Update, dan Delete. Fungsi tersebut merupakan dasar untuk sebagian besar aplikasi software yang berhubungan dengan penyimpanan dan pengambilan data

(2).Menambahkan dokumen baru ke koleksi. Dalam hal ini, Cloud Firestore akan otomatis menghasilkan ID dokumen. Membuat dokumen kosong dengan ID yang otomatis

(3).Di sisi lain, asynchronous mengacu pada proses di mana beberapa tugas dieksekusi secara bersamaan atau sekaligus. Berbeda dengan pemrograman synchronous, program tidak harus menunggu setiap tugas selesai sebelum melanjutkan tugas berikutnya

(4).Langkah 1: Membuat Akun GitHub dan Firebase - Buat akun GitHub jika Anda belum memiliki akun. - Buat akun Firebase jika Anda belum memiliki akun. Langkah (2): Menginstal SPCK yang Diperlukan - Pilih SPCK yang sesuai dengan kebutuhan Anda, seperti React, Angular, atau Vue. - Instal SPCK tersebut menggunakan npm atau yarn. Langkah (3): Membuat Proyek Baru di GitHub - Buat proyek baru di GitHub. - Beri nama proyek dan deskripsi singkat. Langkah (4): Menghubungkan Proyek dengan Firebase - Buat proyek baru di Firebase. - Pilih opsi "Web" dan beri nama proyek. - Salin kode konfigurasi Firebase dan tempelkan di file konfigurasi proyek Anda. Langkah (5): Membuat Struktur Proyek - Buat struktur proyek yang sesuai dengan kebutuhan Anda. - Buat folder untuk komponen, layanan, dan utilitas. Langkah (6): Membuat Komponen dan Layanan - Buat komponen dan layanan yang sesuai dengan kebutuhan Anda. - Gunakan SPCK yang Anda pilih untuk membuat komponen dan layanan. Langkah (7): Mengintegrasikan Firebase dengan Proyek - Gunakan Firebase SDK untuk mengintegrasikan Firebase dengan proyek Anda. - Konfigurasi Firebase untuk menggunakan autentikasi, penyimpanan, dan lain-lain. Langkah (8): Menguji dan Memantau Proyek - Uji proyek Anda untuk memastikan bahwa semua fitur berfungsi dengan baik. - Gunakan alat pemantauan seperti Firebase Analytics untuk memantau kinerja proyek Anda. Langkah 9: Mendebug dan Memperbaiki Proyek - Cari dan perbaiki kesalahan yang ditemukan selama pengujian. - Gunakan alat debugging seperti Chrome DevTools untuk memperbaiki kesalahan. Langkah 10: Mengirimkan Proyek ke GitHub dan Firebase - Kirimkan perubahan Anda ke GitHub. - Deploy proyek Anda ke Firebase Hosting. Langkah 1: Membuat Akun GitHub dan Firebase - Buat akun GitHub jika Anda belum memiliki akun. - Buat akun Firebase jika Anda belum memiliki akun.
